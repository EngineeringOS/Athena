# Epic 2 Retrospective: Native Symbol Library Foundation

## Epic Summary

Epic 2 introduced the first M30 native representation library:

- Story 2.1 created the native symbol asset format and JVM-only loader.
- Story 2.2 added the focused industrial control demo symbol pack.
- Story 2.3 added structural quality tests for anchors, labels, bounds, lifecycle, versions, and deterministic primitive emission.

## What Worked

- The native library is data-driven and does not hard-code symbols in the renderer.
- QET `.elmt`, QET XML, Theia snippets, SVG fragments, and `pathData` are explicitly blocked as runtime source truth.
- The pack stayed focused at eight demo symbols instead of expanding into a weak general-purpose IEC catalog.
- Structural tests now catch missing terminal anchors, missing label slots, invalid metadata, empty primitives, and nondeterministic primitive output before rendering.

## What Needed Correction

- Story 2.1 initially did not assert every AC-1 field in the loader test. Review added explicit checks for version, primitive id, terminal number, and label anchor.
- Story 2.1 tests created temporary files without deleting them. Review added cleanup with `deleteIfExists`.
- The v0 format uses `.properties`, not JSON/YAML. This is acceptable only because it is documented as a bootstrap format and avoids adding a parser dependency without a wider build decision.

## Lessons Learned

- Asset loader tests must assert each required field directly, not only successful load.
- Test hygiene includes external temp-file cleanup, not only repository cleanup.
- The accepted architecture is the Representation Definition IR contract, not the current `.properties` syntax.
- QET remains a reference/import source only; runtime symbol assets must be native Athena definitions.

## Next Epic Preparation

Epic 3 must consume the symbol library through binding rules, not by direct renderer selection:

- Policy chooses symbol family, concrete symbol id, variant, fallback behavior, and priority.
- Binding emits Representation Occurrence IR.
- Missing or ambiguous library matches become diagnostics.
- Demo semantic facts must bind to symbols without frontend inference.
