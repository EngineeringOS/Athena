# Epic 5 Retrospective: Renderer Integration And Projection Controls

## Epic Summary

Epic 5 moved M30 from representation facts to product-visible renderer behavior:

- Story 5.1 rendered native representation primitives from governed Presentation IR.
- Story 5.2 enforced transparent normal chrome so component backgrounds and control docks stopped acting like generic UI cards.
- Story 5.3 made the SVG viewBox derive from live content bounds and blocked duplicate off-sheet render occurrences.
- Story 5.4 preserved governed projection/sheet controls and closed the contract drift between story text and executable smoke.

## What Worked

- Representation rendering stayed upstream-owned: the renderer consumed Presentation IR rather than inventing fallback symbol geometry.
- Transparent normal-state chrome removed the double-border and empty-wrapper failure mode that had damaged M27 credibility.
- Structural SVG guards caught the exact regressions the user had raised before: oversized viewBox, duplicate off-screen renderings, and visible wrapper chrome.
- Projection and sheet controls now remain fact-driven rather than inferring state from source file count.

## What Needed Discipline

- Story artifacts drifted from executable product truth when Cabinet default replaced the older Documentation-default assumption.
- Product smoke was only trustworthy once the Outline proof stopped snapshotting the tree before it populated.
- Renderer work must keep using structural proof and product smoke together; either one alone misses real regressions.

## Lessons Learned

- Product contracts must be taken from the live smoke path, not from stale story text.
- Outline and interaction smoke hooks need explicit waits around async model population.
- View-mode and sheet-selector behavior should stay governed by projection facts, never by editor/source heuristics.
- Wrapperless transparent normal state is not styling polish; it is part of the product contract for dense industrial readability.
