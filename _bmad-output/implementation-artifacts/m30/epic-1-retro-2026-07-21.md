# Epic 1 Retrospective: Representation Contract Foundation

## Epic Summary

Epic 1 established the M30 representation foundation:

- Story 1.1 defined platform-owned representation policy, definition, occurrence, lifecycle, reference binding, diagnostics, and deterministic transport contracts.
- Story 1.2 published the semantic contract audit for representation binding, including allowed binding facts, forbidden kernel concepts, and domain-contract gaps.
- Story 1.3 added stable diagnostic validation and deterministic JSON-safe diagnostic payload tests.

## What Worked

- The representation boundary stayed platform-owned and frontend-independent.
- Definition IR and Occurrence IR were kept separate, which prevents reusable symbol assets from being confused with projected usages.
- The audit explicitly rejected renderer-side meaning inference from SVG, QET data, hitboxes, style tokens, view boxes, and frontend wrapper rules.
- Diagnostic validation now fails explicit fallback policies when a required symbol is missing, so renderer fallback cannot silently pass a proof.

## What Did Not Work

- Story 1.1 was initially marked review while missing an AC-3 field: representation policy had `symbolId` but no explicit `symbolFamilyId`.
- Story 1.2 initially mentioned terminal numbering mostly as a gap, while AC-1 required terminal number to be listed as a binding fact.
- These were specification-to-implementation traceability misses, not test infrastructure failures.

## Lessons Learned

- Every story review must map each Acceptance Criterion to an actual contract field, document line, or executable assertion before status becomes `done`.
- Contract stories need negative review checks for omitted fields, not only positive object construction tests.
- Audit stories must distinguish "available today", "partially available", and "missing domain contract" instead of flattening all three into one narrative.
- The mandatory polish/purge gate is useful only when it includes semantic stale claims, not only dead files.

## Next Epic Preparation

Epic 2 should not only load native symbols. It must preserve the Epic 1 split:

- Symbol assets are Representation Definition IR.
- Policy selects family, variant, role, fallback, and priority.
- Occurrences bind semantic identities and projection context to definitions.
- Any missing domain fact becomes a diagnostic or ledger gap, not renderer inference.
