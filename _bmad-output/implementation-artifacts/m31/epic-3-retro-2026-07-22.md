# M31 Epic 3 Retrospective: Stable Professional Two-Sheet Projection

## Epic Review

### Outcome

Epic 3 completed all three stories. Athena now projects the M31 customer model into exactly two
policy-owned sheets, preserves Cabinet as the default view, derives representation/composition and
routing from semantic state, emits first-class cross-sheet reference evidence, and preserves stable
sheet, occurrence, reference, route, and reveal identities across recompile and reopen.

- Aaron, Project Lead
- Amelia, Developer
- Winston, Architect

### What Worked

- Typed sheet-policy evidence stopped the recurring source-file-count and label-parsing mistakes.
- Re-derivation tests caught the exact visual failures that made earlier milestones look toy-like:
  duplicated off-sheet nodes, fixed oversized viewBox, center fallback routes, generic fallback
  boxes, and repeated labels.
- Keeping Composition Target as intent, not geometry, allowed professional document behavior without
  source-level coordinates.
- Cross-sheet references became semantic occurrences instead of label-text navigation tricks.

### Challenges And Root Causes

- Older M26/M19 reference fixtures were close enough to current behavior to confuse cleanup review.
  The root problem was unversioned compatibility tests that looked like product truth.
- Sheet selector behavior crossed frontend state, runtime payloads, and projection policy. Tests had
  to cover mode switching and not just the initial payload.
- Renderer cleanliness required upstream proof. If projection payloads allow generic boxes or center
  anchors, CSS cannot make the product look professional.

### Key Lessons

1. Sheet count, role, and identity must be policy-owned and typed; source files and UI labels are
   not document authority.
2. Visual density bugs are usually upstream representation/projection bugs, not final CSS bugs.
3. Cross-sheet navigation must carry semantic occurrence ids and governed locations, never depend on
   display text.
4. Legacy fixture names must be either removed or explicitly versioned so future stories do not
   mistake defensive tests for accepted product behavior.

### Quality Evidence

- Stories 3.1, 3.2, and 3.3 are marked done with AC-to-evidence mapping and polish/purge results.
- M31 compiler/runtime/LSP/frontend tests assert exactly two governed sheets, typed policy evidence,
  stable references, no `_reference` production components, anchored routes, and derived bounds.
- Story 5.3 reclassified M31-CL-010 and M31-CL-011 as versioned legacy compatibility, with M31
  product proof independent of those fallbacks.
