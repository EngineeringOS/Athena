# Epic 2 Retrospective - M3 - 2026-07-07

## Epic

- Epic 2: Prove External Domains Can Live Outside The Kernel

## Outcome

- Epic 2 is complete.
- `domain-electrical`, `domain-dummy`, and `examples/m3/` now provide concrete proof that hosted external domains can participate through the same stable contracts.

## What Went Well

- The electrical proof domain was tightened to a narrow, architecture-focused vocabulary instead of growing into product scope.
- The dummy proof domain successfully exercised the same SPI without kernel special cases.
- Runtime inspection, render metadata, and the M3 proof corpus made hosted participation visible to reviewers.
- The final Java 25 build stayed green after the hosted-domain mismatch fix.

## What Hurt

- The compiler originally treated “a lowerer exists” as equivalent to “the active hosted set claimed the authored semantics.”
- Dummy-domain documentation drifted away from the actual authored syntax.
- Epic 2 needed one more review/fix loop before it could be truthfully marked done.

## Key Lessons

1. Hosted-plugin presence is not enough; the compiler must detect when no active plugin actually claimed the authored declarations.
2. Proof-domain documentation must be validated against real fixtures, not inferred from intent.
3. The published proof corpus is the right seed for Epic 3 matrix automation.

## Follow-Through For Next Epic

- Start Epic 3 from the corrected baseline, not from the earlier review state.
- Reuse `examples/m3/` directly for zero/one/multi-plugin verification.
- Keep backend blocking behavior honest when a domain has no global default view definitions.

## Action Items

1. Use the new mismatch regression as the guardrail for further hosted-domain work.
2. Make the Epic 3 matrix the first-class proof target, not an afterthought.
