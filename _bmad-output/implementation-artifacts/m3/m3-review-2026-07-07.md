## Findings

### 1. High - Wrong hosted plugin set can silently pass semantic validation with an empty canonical model

- Files:
  - `kernel/compiler/src/main/kotlin/com/engineeringood/athena/compiler/AthenaCompiler.kt:744`
  - `kernel/compiler/src/main/kotlin/com/engineeringood/athena/compiler/plugin/AthenaDomainSemanticsCoordinator.kt:64`
- `domainSemanticsUnavailableDiagnostics()` suppresses `domain.semantics.unavailable` as soon as *any* lowering participant exists.
- `AthenaDomainSemanticsCoordinator.hasParticipants(LOWER)` only answers whether a plugin is present for the stage, not whether that plugin actually claimed any authored declarations.
- The lowering coordinator can still return `AthenaDomainLoweringContribution.EMPTY` for a mismatched hosted set, because `lower()` just folds plugin contributions and does not report uncovered authored declarations.
- Concrete risk:
  - a document authored with electrical declarations compiled under `dummy-only` hosted inventory can pass the “domain semantics unavailable” gate even if no declaration was claimed by the dummy plugin
  - the result is an empty `Engineering IR` plus false semantic success instead of a loud mismatch diagnostic
- Why this matters:
  - it weakens AD-1, AD-2, and AD-6
  - it hides plugin-set mistakes behind apparently valid compiler output
- Recommended fix:
  - track authored declaration coverage during lowering, or
  - add a post-lowering mismatch rule that emits `domain.semantics.unavailable` when authored declarations exist but no active plugin claimed them
  - add a regression for `electrical-authored source + dummy-only inventory`

### 2. Medium - Dummy-domain README documents DSL syntax the current parser does not accept

- File:
  - `extensions/domain-dummy/README.md:20`
- The README says the ownership marker is ``domain dummy-runtime``.
- Current authored fixtures and story notes use ``domain "dummy-runtime"`` because the grammar does not accept the hyphenated bare identifier.
- A developer following the README will write invalid source even though the checked-in examples use the correct quoted form.
- Recommended fix:
  - change the README to ``domain "dummy-runtime"``
  - apply the same correction to `extensions/domain-dummy/README.zh-CN.md`
