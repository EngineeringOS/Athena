# M32 Epic 4 Retrospective: Package-Backed Representation Integration

## Scope

Epic 4 implementation reached `review` for:

- Story 4.1: Feed descriptors into Representation Occurrences
- Story 4.2: Render Graphic Resources as descriptor-backed resources
- Story 4.3: Route through descriptor anchors

This is an implementation retrospective. Story statuses remain `review` until code review promotes
them.

## Epic Review

Epic 4 connected the package resolver and binding evidence layer into the M30 representation path:

- Package-backed occurrences now bridge descriptor facts into the existing
  `RepresentationBindingCompiler` / `RepresentationOccurrence` model.
- Descriptor-backed render payloads expose paint-only Graphic Resource handles, bounds, anchors,
  slots, transparency, and fallback guards without making resources semantic authority.
- Descriptor-anchor route evidence maps terminal routes through binding evidence and descriptor
  anchors, and fails closed when anchor facts are missing.
- Product proof data now has a route from resolver facts to occurrence, render, and routing
  evidence.

## What Worked

- Keeping `:kernel:package-runtime` as the integration boundary avoided leaking descriptor loading,
  resource details, or renderer assumptions into `:kernel:package-model`.
- TDD caught each missing bridge as unresolved symbols before implementation, which kept scope
  aligned to the story contracts.
- The descriptor-backed render payload made transparency and duplicate-label proofs explicit
  instead of relying on SVG inspection.
- Route evidence forced the important rule: no center fallback is successful package-backed
  routing.

## What Needs Attention

- Epic 5 must prove the full path with a real M32 sample and product smoke, not just isolated
  runtime tests.
- The Theia renderer is still adapter-only for M32 package payloads. The demo proof must consume
  structured evidence and document any frontend display work deferred beyond M32.
- Descriptor-backed payloads currently use in-memory descriptors. Future file loading still belongs
  behind package-runtime.

## Lessons Learned

1. Representation Occurrence is the right bridge point; adding a parallel package occurrence model
   would fragment M30.
2. Graphic Resource is paint input only. Bounds, anchors, slots, and interaction safety must come
   from representation descriptors and binding evidence.
3. Routing should fail loudly when anchors are absent. A visually convenient center fallback hides
   engineering errors.
4. Product proof payloads are now more important than DOM screenshots for package-layer correctness.
