# M25 Epic 4 Retrospective

Date: 2026-07-19
Milestone: M25 Symbol and Presentation Policy Foundation
Epic: 4 - Product Proof, Usage, and Boundary Closure

## Summary

M25 Epic 4 converted the M25 representation work into an openable product proof. The delivered proof
is not a script-only fixture. It is a Theia sample workspace at `examples/m25/sample-project` with
real `.athena` source files, product launch wiring, Electron smoke coverage, usage documentation,
and acceptance notes comparing the M24 route baseline against M25 governed presentation anatomy.

The most important result is that Athena now proves this chain in the IDE:

```text
component knowledge
  -> presentation policy
  -> Presentation IR representation facts
  -> terminal facts and label facts
  -> M24 route facts
  -> Theia paint-only Graphical View
```

The renderer still does not invent engineering meaning.

## What We Learned

The critical integration lesson is that frontend model tests must cover the real customer-visible
Presentation IR path. Earlier M25 tests covered fallback graph nodes with `representationFacts`, but
the sheet used `presentation.occurrences`. That path was dropping `presentationRepresentation`,
`presentationTerminals`, and `presentationLabels`.

The fix was not to move meaning into the renderer. The correct fix was to preserve upstream
representation facts while building the Theia workbench model:

```text
Presentation IR occurrence
  -> source projection id / subject id
  -> representation fact
  -> scaled presentation parts, terminals, and labels
  -> renderer DOM markers
```

This keeps M25 aligned with Athena's semantic-first architecture.

## Boundary Scan

The M25 implementation and docs were scanned for stale scope claims around:

- QElectroTech import
- `.elmt` ingestion
- full IEC completeness
- full EPLAN parity
- new source syntax
- route-hint syntax expansion
- desktop-viewer, Compose, or deprecated KMP frontend scope

M25-specific artifacts correctly keep these as deferred or excluded. Older cross-milestone usage
docs still mention desktop-viewer or Compose as historical proof paths, but those are not M25 scope
claims and were not rewritten in this story.

## Deferred Work

M25 still does not deliver:

- full IEC symbol library breadth
- QElectroTech `.elmt` import
- symbol authoring UI
- manufacturer or company-standard presentation packs
- full EPLAN visual parity
- physical cabinet routing, harness routing, cable tray routing, or 3D routing
- route editing or route-hint syntax expansion

These remain later milestones.

## Usage

Build the product path and open the M25 sample:

```powershell
.\gradlew.bat --no-daemon --console=plain :ide:lsp:installDist
yarn --cwd ide build
yarn --cwd ide start:m25
```

Run the product smoke:

```powershell
yarn --cwd ide start:smoke:m25
```

Run the regression proof:

```powershell
yarn --cwd ide start:smoke:m24
yarn --cwd ide/theia-frontend test --test-name-pattern "Presentation IR occurrences"
```
