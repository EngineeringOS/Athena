### What Worked Well

- The authored-versus-derived distinction stayed intact from diff through review and commit preparation.
- Plugin enrichment remained additive and governed, which preserved the semantic authority boundary.
- Runtime owned the projection state cleanly, so LSP and Theia stayed transport and presentation layers instead of becoming semantic owners.
- The existing Theia panel was sufficient to surface the work; no shell rewrite was needed.

### Challenges And Lessons

- Transport payloads should mirror runtime-owned types closely. When the projection layer drifts, tests catch it quickly and the fix is straightforward.
- Theia build validation is necessary even when the Kotlin side is green; JSX and TypeScript narrowing issues surface only in the frontend toolchain.
- Review and commit output are much easier to reason about when each entry remains traceable back to stable semantic facts.

### Technical Debt And Residual Risk

- There is still no automated interactive IDE E2E coverage for the review and commit parts of the semantic SCM panel.
- Hosted plugin enrichment is proven in the governed path, but the broader plugin catalog remains intentionally narrow.
