### What Worked Well

- Splitting family, sheet, notation, and projection nouns into cleaner files made the kernel boundary easier to read and harder to blur.
- Reusing the existing runtime and LSP seams prevented view depth from creating a second product-owned authority.
- Separating `semanticId` from projection-local occurrence ids kept reveal coherence honest.

### Challenges And Lessons

- M11 only became credible once notation, sheet, and view-family data were all treated as downstream contracts instead of ad hoc payload fields.
- Toy projection cases were not enough to validate identity behavior; even Epic 1 needed denser cross-surface reasoning than earlier milestones.
- Windows verification discipline is still non-negotiable: Java 25 activation and sequential Gradle execution remain mandatory.

### Technical Debt And Residual Risk

- The electrical vocabulary is still intentionally narrow and not yet a broad ECAD domain model.
- The workbench consumes the richer view contracts, but product-level UX depth remains later than this epic.
- Desktop E2E automation still does not prove the full human-facing multi-view flow.

## Next Epic Preparation
