- (decision) Architecture paradigm is package-resolved representation: semantic source resolves package descriptors and policy before presentation or renderer resources.
- (constraint) M27-M31 invariants remain binding: mutation authority, semantic interaction, authoring transactions, presentation bounds, transparent normal chrome, and no downstream artifact mutation.
- (decision) M31 deferred compatibility debts become explicit M32 cleanup work with evidence gates rather than untracked background tasks.
- (decision) Engineering Package descriptors own product and concept facts but no rendering resources; Representation Package descriptors own resource contracts but no engineering truth.
- (decision) Binding Manifest and Binding Policy are the only bridge from engineering package facts to representation package choices; renderer cannot infer binding from resources.
- (decision) Local registry discovery is deterministic and governed by explicit roots; no arbitrary workspace scan or internet registry is part of M32.
- (decision) QET is a deferred offline import research source only; no runtime dependency and no direct .elmt to .athena conversion.
- (decision) Presentation Profile is an independent package/policy artifact for IEC, ANSI, customer, print, maintenance, compact, training, and theme choices; it owns neither engineering truth nor graphic resources.
- (decision) Binding Resolver becomes a named compiler stage between Engineering Resolver and Representation Resolver to support deterministic and incremental package binding.
