- (decision) AD-1: Use a semantic action pipeline paradigm; Interaction IR carries human-facing behavior while Semantic Action Intent is the producer-neutral primitive.
- (decision) AD-2: Create a platform-owned interaction-model boundary outside Theia; frontend adapters may consume but not own canonical interaction meaning.
- (decision) AD-3: Build a Semantic Capability Registry from semantic identities and projection facts; it is not a frontend lookup map.
- (decision) AD-4: Introduce Interaction Compiler v0 as derivation only; it discovers actions and payloads but does not mutate source or projection state.
- (decision) AD-5: Route accepted relationship/entity creation commands through existing authoring/runtime mutation authority and source serialization gates.
- (decision) AD-6: Treat Interaction Runtime as session/lifecycle/provenance state, not durable semantic truth.
- (decision) AD-7: Theia/LSP/GLSP are adapters and transports; DOM text, SVG geometry, CSS classes, and widget ids cannot become semantic authority.
- (decision) AD-8: Semantic entity creation creates model entities first; symbol placement is a projection consequence.
- (decision) AD-9: M29 preserves standards/presentation metadata but defers IEC/QET/EPLAN library and visual parity work.
- (decision) AD-10: Product proof uses structured payload assertions before UI click assertions, and no Gradle verification runs concurrently.
