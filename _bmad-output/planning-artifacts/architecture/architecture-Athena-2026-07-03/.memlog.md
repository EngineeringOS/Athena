- (event) Architecture create run started from the draft Athena M1 PRD on the fast path with the spine itself treated as the main deliverable for builders.
- (constraint) Inherited M0 invariants remain binding unless explicitly superseded: general core semantics, Engineering IR as canonical semantic authority, rendering downstream of semantics, plugins non-sovereign and locally governed, and examples as conformance artifacts.
- (decision) M1 paradigm chosen: single-process layered semantic runtime with command-sourced mutation and compiler/render services hosted behind runtime-owned contracts, preventing the compiler from remaining the operational center of gravity.
- (decision) Athena Runtime becomes the sole owner of workspace lifecycle, project activation, execution context, and service registry, while compiler functionality is consumed as a runtime service rather than the root orchestrator.
- (decision) Engineering IR remains the only canonical semantic authority in M1; engineering graph, command history, viewer state, and plugin runtime are projections or mutation channels over canonical semantics, not competing durable models.
- (decision) All semantic mutation in interactive and non-interactive surfaces flows through the command runtime, preventing direct UI, AI, or caller-side mutation of canonical project state.
- (decision) Engineering Graph is a runtime-owned graph projection over project semantics with shared stable identities and dependency traversal, preventing drift into a second semantic authority.
- (decision) Frontends remain adapters only: DSL, GUI, and AI all enter Athena through runtime contracts and converge on the same command, validation, and rendering path.
- (decision) Compose runtime is the first interactive viewer substrate for M1 and remains domain-neutral infrastructure for viewport, selection, input, and camera behavior, preventing electrical semantics from leaking into the UI runtime.
- (decision) Incremental validation and rendering are runtime-triggered and dependency-scoped: changes identify affected semantic scope first, then re-run only the required graph, validation, and rendering work.
- (decision) Plugin Runtime v2 is hosted by Athena Runtime and may contribute commands, views, importers, exporters, AI skills, and domain semantics without taking ownership of Engineering IR, lifecycle, or orchestration.
- (version) Reality-checked seed stack from the current repository: Java 25, Kotlin 2.4.0, Gradle 9.6.1.
- (version) Compose Multiplatform pinned for the first M1 viewer target at 1.11.0 after author-time JetBrains release check on 2026-07-03.
- (direction) Migration direction is evolutionary rather than rewrite-driven: add runtime above proven M0 modules first, then extract compiler-adjacent ownership only where the new owner is explicit.
