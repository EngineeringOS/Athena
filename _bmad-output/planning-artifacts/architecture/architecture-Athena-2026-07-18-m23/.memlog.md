- (decision) M23 admits only system-scoped layout blocks first; file-global/project/company layout policy stays deferred.
- (decision) Generated parser types remain internal; Athena-owned LayoutDeclaration AST is the handoff into compiler and source tooling.
- (decision) LayoutDeclaration lowers to layout intent before constraints; AST stays syntax-only and compiler owns binding/diagnostics.
- (decision) ANTLR4 and Tree-sitter parser parity is mandatory; compiler/LSP parser and IDE syntax parser must accept the same M23 layout syntax.
- (decision) Layout priority is model-owned in M23; authored statements default to preference unless a later source keyword is admitted deliberately.
- (decision) Graph Workbench emits layout intent and uses an Athena serializer/source-edit path; frontend code must not be the syntax authority.
- (constraint) M23 preserves M0-M22 syntax compatibility, M18 package/import behavior, and M20-M22 active-source Graphical View/canvas behavior.
- (constraint) M23 excludes EPLAN parity, advanced routing, AI layout, public repository/import ecosystem, full IEC library work, and hidden canvas-state persistence.
